Using Chrome Developers Tool

Calling yourself a super web developer and don't know about Chrome Devtool.
Today, will go through how we can enhance our productivity using chrome devtools.

Chrome DevTools is a set of web developer tools built directly into the Google Chrome browser.

DevTools can help you edit pages on-the-fly and diagnose problems quickly, which ultimately helps you build better websites, faster.Here is an example of editing google home page.

Discover DevTools

To start using dev tool press F12 or Ctrl + Shift + i in you chrome new tab or web page which you want to inspect.Now you will find a number of tabs there.Will explain you some of handy ones here.

On a journey to become a super developer

Device mode Are you building a responsive website and want to test it on different devices, go buy them all . Not a good idea, you can make use of chrome device mode to simulate different device and check responsiveness, just click on Toogle Device Toolbar option on top left or press Ctrl + Shift + M

Testing Responsiviness of website in device mode

Elements Using this tab you can add or edit html code or make changes in CSS and can check its reflection right there at the same time.It become very handy for making CSS changes in a website as all the changes are reflected immediately
Console In this tab, you will get all the javascript console logs.And thats not all, you can execute any javascript code directly here in console and advance features like autocomplete are also integrated with this console.There is also list of command line API's build in console for common task details of which you can find here

Debugging javascript using chrome devtool

Source Using this tab, you can debug you javascript source code, view call stack and all your files.You can even persist you code changes while navigating to different pages using local over-rides and save your changes to disk
Network Web developers generally get confused with network error and is difficult to debug as they don't know the error is from server side or by missing any param while calling API.To make it a bit easy, you can use network tab to get the details about you request and response through network which helps you to find out the source of error.

Network details in network tab

Application Building a PWA, must learn to use this.You can check you service worker and manifest file.Apart from that you can also view different storages like local storage, session storage, cache or cookies of your website.
Code Coverage Using this you can check the code coverage of you javascript or CSS file. It gives a very detailed representation of each line of your code so that you can refactor you code to make it more effiecient and remove unused code/css from your file.

Audits This is used to check the performance of your website.It internally uses Lighthouse which is analyzes web apps and web pages, collecting modern performance metrics and insights on developer best practices.

That's not all, there is a lot more to explore in chrome devtool and it keeps on getting evolved with each new release of chrome which bring awsome new features to help developers increase their productivity and build awsome apps for billions to use.
