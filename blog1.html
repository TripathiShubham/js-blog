Future of Web: Progressive Web App

Have you ever seen an “Add to Home Screen” banner, like above, while browsing a website? When you click the button, the “application” installs itself in the background. When you open this application that now sits in your app drawer, you can browse the same experience you were doing on your browser, but now right in your mobile phone.

This, we call as Progressive Web App (PWA).

A Progressive Web App uses modern web capabilities to deliver an app-like user experience.

Browser's are awsome, we are using browsers for surfing the internet from year's. We just need to enter the url and the its on your door step.The capabilities of browsers evolved from decades.Browsers now are just not serving html files but much more than that.And PWA is one of the latest feature of Web.

Why build a Progressive Web App?

PWA is set of best practices to build web apps which feels like native apps on desktop or mobile.

User engagement on web vs mobile

PWA fill's this gap between the mobile app and web app with the following features.

Reliable We love to have downasaur but not in our browser screen, PWA works even offline and provide us seamless experience online or offline.
Fast Loads instantly with fluide user experience.
Engaging Feel like a natural app on the device, with an immersive user experience and can even re-engage users with web push notifications.

PWA with offline, add to home screen and push notification

Components of a PWA

Implement app shell which is the basic minimal HTML, CSS, and JavaScript that is required to power the user interface instantly.
Use Service Worker to intercept network request and cache information for the app which can be used to serve when offline.Here is sample service worker for your refference. Add manifest file which is a JSON file used by browser to give the developer the ability for that app to be added to your home screen and control how your app will appear.
